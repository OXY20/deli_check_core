# 文档目录

## 版本发布记录

### v1.2.0（2026-05-07）

#### 修复问题
- **单天考勤解析失败**：当导出的考勤记录仅一天时，原 `findBaseDate` 正则仅匹配带范围格式（`考勤日期：2026-01-04～2026-01-31`），无法识别单日期格式，已新增 `baseDateSingleRe` 正则解决。
- **非1起始日期表头识别失败**：原 `detectDayHeaderRow` 要求日期表头必须从 `1` 开始（`1,2,3...`），部分月考勤（如 `15,16,17...`）直接被忽略，已改为宽松的连续递增校验。
- **数据行误判为日期表头**：单天模式下，仅含一个整数的行可能被数据行中的纯数字（如加班时长）误判，已改为要求行中**所有非空单元格均为纯数字**方可认定为日期表头。
- **死代码清理**：移除 `detectDayHeaderRow` 中不再使用的 `startCol = -1` 和 `expected := 1` 变量声明。

#### 新增功能
- **自动化测试**：`tools/excel_test.go` 新增 36 个测试用例，覆盖解析全流程关键路径，支持 CI 回归验证。

### v1.1.0（2026-04-14）

#### 新增功能
- **多表信息合并模式**：支持同时上传多个 `.xls` / `.xlsx` 考勤文件
  - 每个文件可自定义地点名称（最多 8 个字符）
  - 保留所有原始打卡记录，按人员统一展示
  - 人员显示表格新增「地点」列，超长名称自动截断并支持悬浮提示
  - 用餐统计仍按原有逻辑去重（同一天同一时段计 1 次）
- **导出人员数据**：新增蓝色导出按钮，支持导出人员明细 `.xlsx`（多表模式下含「地点」列）

### v1.0.0（2026-04-14）

#### 核心功能
- 支持 `.xls` / `.xlsx` 双格式考勤文件解析
- 自动识别员工信息块，动态提取工号、姓名、部门
- 跨行数据自动合并解析
- 单文件 / 目录批量扫描两种模式
- 输出结构化 JSON（`records.json`、`summary.json`）

#### Web 界面（`deli_check_web` 分支）
- 浏览器端上传考勤文件
- 按人员查看打卡记录详情
- 用餐统计功能：按时段统计早餐 / 午餐 / 晚餐次数
- 导出用餐数据为 `.xlsx` 表格
- 支持开发模式热重载（`DELI_WEB_DEV=true`）

#### 分支说明
- `main`：CLI 核心解析工具主分支
- `deli_core`：核心解析分支（与 `main` 同步）
- `deli_check_web`：Web 界面主基线分支
- `develop_deli_check_web`：Web 界面日常开发分支

此目录用于存放项目相关文档和说明。
